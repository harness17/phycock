# Phycock 要件定義書

## プロジェクト概要

**プロダクト名**: Phycock（ピーコック） — 体調管理アプリ  
**対象ユーザー**: リタリコワークス通所者（当面はダミーデータでのポートフォリオ運用）  
**ビジネス目的**: 通所者の体調管理・スケジュール管理・支援者への情報提供を効率化  
**開発基盤**: Phycock（ASP.NET Core 10 MVC）  
**スケジュール**: フェーズ別実装

## ユースケース

### UC-1: 体調記録

**アクター**: 通所者（Member）

**前提条件**: ログイン済み

**フロー**:
1. ダッシュボードから「体調記録」を選択
2. 本日の日付が自動入力される
3. 症状をチェックボックスから選択（複数選択可）
4. 体調・気分をラジオボタンから選択
5. メモを記述
6. 「保存」ボタンをクリック
7. 確認メッセージ表示 → 一覧画面に遷移

**代替フロー**: 過去の記録を編集する場合は、一覧から該当日付をクリック

**例外処理**:
- 必須項目未入力 → バリデーションエラーを赤字表示
- DB エラー → 「エラーが発生しました。しばらく後に再試行してください」と表示

---

### UC-2: スケジュール参照

**アクター**: 通所者（Member）/ スタッフ（Admin）

**前提条件**: ログイン済み

**フロー**:
1. ダッシュボードからカレンダーアイコンをクリック
2. FullCalendar で現在月を表示
3. 予定をクリック → 詳細ポップアップ表示
4. 予定を編集 → 保存

**例外処理**:
- 予定がない日付 → 「予定なし」と表示

---

### UC-3: ユーザー管理（Admin）

**アクター**: スタッフ（Admin）

**前提条件**: Admin ロールでログイン済み

**フロー**:
1. ダッシュボード → 「ユーザー管理」メニュー
2. ユーザー一覧を表示
3. 「ユーザー追加」ボタン → フォームを表示
4. メール・パスワードを入力
5. ロール（Member / Admin）を選択
6. 「作成」ボタン → 確認メッセージ

**例外処理**:
- メールアドレス重複 → 「既に登録済みです」
- パスワード不足 → パスワード規則（6 文字以上、大小文字・数字・記号含む）を表示

---

### UC-4: 睡眠時間記録

**アクター**: 通所者（Member）

**前提条件**: ログイン済み

**フロー**:
1. ダッシュボードから「睡眠時間記録」を選択
2. 本日の日付が自動入力される
3. 睡眠・仮眠の区分け、睡眠時間を入力
4. メモを記述
5. 「保存」ボタンをクリック
6. 確認メッセージ表示 → 一覧画面に遷移

**代替フロー**: 過去の記録を編集する場合は、一覧から該当日付をクリック

**例外処理**:
- 必須項目未入力 → バリデーションエラーを赤字表示
- DB エラー → 「エラーが発生しました。しばらく後に再試行してください」と表示

### エンティティベース

ASP.NET Core Identity関連以外のエンティティは `EntityBase` を継承：
```csharp
public abstract class EntityBase
{
    public long Id { get; set; }
    public long CreatedBy { get; set; }
    public DateTime CreatedAt { get; set; }
    public long? UpdatedBy { get; set; }
    public DateTime? UpdatedAt { get; set; }
}
```

### 列挙型定義

#### 体調・気分スコア（Condition / Feeling）

| 値 | ラベル |
|----|--------|
| 1 | 良い |
| 2 | やや良い |
| 3 | 普通 |
| 4 | やや悪い |
| 5 | 悪い |

#### 症状（Symptoms）暫定リスト

症状はカンマ区切りの文字列で保存。以下は暫定リスト（スタッフ確認後に更新予定）。

**体調面（7種）**
頭痛 / 腹痛 / 吐き気 / めまい / 発熱 / 食欲不振 / 動悸

**精神面（6種）**
気分の落ち込み / 不安感 / イライラ / やる気の低下 / 集中力の低下 / 焦り

**感覚面（5種）**
眠気 / 倦怠感 / 体の重さ / 筋肉の緊張 / 感覚過敏（音・光）

## セキュリティ要件

| 要件 | 対応方法 |
|------|---------|
| 認証 | ASP.NET Core Identity（メール + パスワード） |
| 認可 | ロールベース（Admin / Member） |
| IDOR 対策 | 自分のデータのみ閲覧・編集可（Admin は全員） |
| パスワードポリシー | 6 文字以上、大小文字・数字・記号必須 |
| セッション管理 | cookie ベース、タイムアウト 30 分 |
| HTTPS | 本番環境で必須、HTTP → HTTPS リダイレクト |
| エラーメッセージ | DB スキーマ・内部情報を露出しない |
| 監査ログ | CreatedBy / UpdatedBy で全操作を追跡 |

## 質問・備考
ユーザー数の予測: 初期 10-20 名

モバイル対応の優先度: Phase 1 

体調報告は起床時、訓練開始時、訓練終了時、就眠時に入力できるようにする。(全て入力しきる必要はない)

睡眠時間の記録は一日に複数入力できるようにすること。

ダッシュボードでの体調グラフは週・月単位でも確認可能なように。

スケジュールの登録単位は一日。基本的にはリタリコの通所・在宅スケジュールに合わせる。
リタリコの通所時間は基本的に9:00～12:00 13:00～15:00 左記2つの組み合わせがある (10:00からや外出などの変則スケジュールもある。内容も種類があるのでここは詰めて行きたい。)

通所内容としては「プログラム(6種のテキストからなる授業)、個別訓練、部署活動、実習、外出、面談、プライベート、その他」が分類としてある。その他は自由入力化。

スケジュール・体調をカレンダー状にエクセルやPDF出力できると良い。
データとしての出力も必要。


スタンドアローンのアプリ版もあるといいかも。これは今後課題。
