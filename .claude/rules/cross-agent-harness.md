# Cross-Agent Harness

Codex と Claude Code など、複数の開発エージェントが同じリポジトリで作業するときの汎用ハーネス。

このファイルはプロジェクト非依存の本体として扱う。技術スタック、verify コマンド、レビュー観点、担当境界は `project-collaboration-profile.md` で定義する。

## 目的

- 実装者とレビュー担当を分け、片側視点の見落としを減らす。
- 触ってよい範囲、完成条件、verify、レビュー結果を同じ handoff に残す。
- merge 前に最低限のゲートを揃え、未レビューの変更が安定ブランチへ入ることを防ぐ。

## 汎用役割

| フェーズ | 既定担当 | 説明 |
|---------|----------|------|
| 仕様・設計 | Claude Code | ユーザー対話、要件整理、未確定事項の収束 |
| 実装（小〜中タスク） | Codex | スコープが明確な実装、テスト追加、局所修正 |
| 実装（広範囲・契約変更） | Claude Code | 複数モジュール、永続化、外部契約、運用判断を伴う変更 |
| レビュー | 作成者の反対側 | 動作、契約、テスト、セキュリティ、スコープを確認 |
| 実動確認 | project profile で指定 | UI、ブラウザ、デスクトップアプリ、外部サービスなどの確認 |
| handoff 更新 | 作業した側 | 依頼、実装結果、verify、レビュー結果を作業直後に残す |

## タスク振り分け

次の観点で `project-collaboration-profile.md` の担当境界に従う。

- 仕様が明確で、触るファイルが限定されるか
- 永続化 schema、API contract、権限モデル、外部サービス契約を変えるか
- 実動確認の主担当がどちらか
- 作業範囲が他エージェントやユーザーの未コミット変更と重なるか
- 完成条件を客観的に書けるか

仕様や完成条件が曖昧なまま、実装担当へ渡さない。

## Merge ゲート 4 条件

安定ブランチへ merge する前に 4 条件すべてを揃える。

| # | 条件 | 確認方法 | 担当 |
|---|------|---------|------|
| 1 | セルフ verify | project profile の verify コマンドが pass | 実装者 |
| 2 | 相互レビュー記録 | handoff にレビュー結果が残っている | レビュー側 |
| 3 | 重大指摘なし | 重大指摘が解消済み、または明示的に延期判断済み | 実装者・レビュー側 |
| 4 | ユーザー merge 指示 | ユーザーが明示的に merge OK と言った | ユーザー |

## 指摘ラベル

- 重大: merge ブロッカー。動作不良、セキュリティ、契約違反、データ破壊、テスト失敗、既存機能破壊、依頼範囲違反。
- 軽微: merge 可否を止めない。命名、コメント、整形、読みやすさ、文書表現。
- 良好: 問題なし。

## スプリントコントラクト

実装前に完成条件を明文化する。最低限、次を含める。

- 通常動作
- 利用前提・認可・環境前提
- エラー処理
- 非回帰確認
- verify コマンド

## スコープ管理

- 触ってよい範囲と触ってはいけない範囲を handoff に書く。
- 依頼外ファイルへ変更が必要になったら、変更前に handoff またはユーザーへ理由を明示する。
- 他者の未コミット変更を戻さない。
- commit / merge / push の担当は handoff に明記する。
