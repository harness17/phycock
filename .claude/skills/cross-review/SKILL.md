# /cross-review

共同開発の相互レビュースキル。

## 起動条件

- Codex が実装を完了し、ハンドオフのセルフ verify が ✅ になったとき
- ユーザーが「レビューして」「クロスレビュー」と言ったとき

## レビュー観点

各観点で重大 / 軽微 / 良好を判定する。具体的な観点は `.claude/rules/project-collaboration-profile.md` を優先する。

### 観点 1: 動作

- 完成条件を満たしているか
- 既存機能を壊していないか
- エラー処理・境界値の扱いに穴がないか

### 観点 2: 契約

- project profile で定義された API、UI、DB、外部サービスなどの契約が呼び出し側と一致しているか

### 観点 3: テスト

- 完成条件に対応するテストが追加・更新されているか
- 正常系、異常系、境界値、認可、重複操作を必要に応じて確認しているか
- project profile の verify コマンドが pass するか

### 観点 4: セキュリティ・監査

- project profile で定義されたセキュリティ・監査観点を満たしているか

### 観点 5: スタイル

- 既存のコードスタイルに揃っているか
- コメントが過剰でないか
- unrelated cleanup や依頼外ファイル変更が混ざっていないか

## 手順

### 1. ハンドオフを読む

`CLAUDE_CODE_HANDOFF.md` の最新セクションから対象・変更ファイル・完成条件・レビュー観点を確認する。

### 2. 変更ファイルを読む

対象ファイルを順に読み、project profile の観点でチェックする。

### 3. 既存テストを実行

`project-collaboration-profile.md` の verify コマンドを実行する。

失敗するなら重大指摘にする。

### 4. 実動確認が必要ならブラウザ確認

project profile で実動確認が必要とされる変更なら、指定された方法で確認する。

### 5. ハンドオフにレビュー結果を追記

`.claude/rules/handoff-protocol.md` の書式に従い、「### レビュー結果（YYYY-MM-DD, レビュー側）」セクションを該当セクションに追記する。

### 6. ユーザーに報告

- 公開可否（良好 / 修正必須 / 軽微指摘あり）
- 重大指摘の要約
- 次アクション（Codex に修正を返すか、ユーザー merge 指示待ちか）

## 注意

- project profile の重大指摘に該当するものは merge ブロッカーとして扱う。
- 動作確認なしで UI 変更を良好にしない。
- 主観的な読みやすさだけの指摘は軽微に留める。
