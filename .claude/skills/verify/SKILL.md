# /verify — ビルド・テスト・検証スキル

実装完了後、本番環境へのデプロイまたはマージ前に実行してください。

## 実行内容

### 1. ビルド確認
```bash
dotnet build --no-restore
```

**確認項目:**
- [ ] エラーなくビルド成功した
- [ ] 警告（CS警告）がないか確認（実装依存で許容可）

### 2. テスト実行
```bash
dotnet test --no-build
```

**確認項目:**
- [ ] すべてのテストが PASS した
- [ ] カバレッジが目標値（80% 推奨）を満たしているか

### 3. コード規約チェック（手動）

実装したコード範囲について以下を確認：

- [ ] PascalCase / camelCase が統一されているか
- [ ] `using IdentitySignInResult` エイリアスが必要な箇所で使用されているか
- [ ] エラーメッセージに DB スキーマ・内部情報が露出していないか
- [ ] IDOR（水平権限昇格）対策がされているか（認可チェック）

### 4. セキュリティチェック

- [ ] APIキー・接続文字列が `appsettings.json` に硬書きされていないか
- [ ] 認証チェックと所有権チェックが分離されているか（`[Authorize]` + `if (resource.UserId != currentUserId)` 両方）
- [ ] 論理削除・監査ログ対応が設計通りか

### 5. DB マイグレーション検証

新規エンティティ追加時のみ：

```bash
# 最新マイグレーション確認
dotnet ef migrations list --project Phycock

# 破壊的変更がないか ChangeLog レビュー
```

- [ ] テーブル削除・カラム削除がないか確認
- [ ] 既存データへの互換性が維持されているか

## 実行例

```bash
# Phycock ディレクトリから実行
cd Phycock
dotnet build --no-restore
dotnet test --no-build

# 確認項目をチェック
# ✅ すべて通ったら、マージ / デプロイ準備完了
```

## 失敗時の対応

| エラー | 対応 |
|---|---|
| ビルドエラー | スタックトレースを確認し、該当コード修正 |
| テスト失敗 | テストログで失敗理由を特定し、実装またはテストを修正 |
| セキュリティ警告 | CLAUDE.md の「セキュリティ」セクション参照 |

## デプロイ前チェックリスト

- [ ] /verify すべてが通った
- [ ] DB マイグレーション内容を確認した（新規追加時）
- [ ] コードレビュー完了（複数ファイル変更時）
- [ ] ドキュメント更新済み（API 変更時）
