# Phase 1 MVP 実装計画

**作成日**: 2026-05-03  
**対象フェーズ**: Phase 1 (MVP)  
**プロジェクト**: Phycock — 体調管理アプリ（ASP.NET Core 10 MVC）

---

## スプリントコントラクト（完成条件）

**正常系:**
- [x] Member がログイン後、体調記録を同日複数タイミング（起床時・訓練開始時・訓練終了時・就眠時）で登録できる
- [x] 睡眠記録を一日複数回登録できる（本睡眠＋仮眠など）
- [x] FullCalendarに通所スケジュールが表示され、追加・編集・削除できる
- [x] ダッシュボードに今日の通所予定と体調サマリーが表示される
- [x] 統計ページで週/月単位の体調・睡眠グラフ（Chart.js）が表示される

**認可:**
- [x] Member は他ユーザーの体調記録・睡眠記録・スケジュールを閲覧・編集できない（IDOR対策）
- [x] Admin は全ユーザーの体調記録を閲覧できる
- [x] Admin のみユーザー管理画面にアクセスできる
- [x] 未認証ユーザーは全ページでログイン画面にリダイレクトされる

**異常系:**
- [x] 体調記録・睡眠記録で必須項目未入力時にバリデーションエラーが赤字表示される
- [x] DB エラー時に内部情報を露出しないエラーメッセージが表示される
- [x] 他ユーザーのリソースへのアクセスは HTTP 403 Forbid を返す

**副作用:**
- [x] 既存の AccountController・ScheduleController・UserManagementController は動作を維持する
- [x] 既存マイグレーション（20260406122823_InitialCreate）との互換性を維持する

## スコープ

### やる（Phase 1）
1. HealthRecord（体調記録）CRUD
2. SleepRecord（睡眠時間記録）CRUD
3. ScheduleEntry（通所スケジュール）新設 + FullCalendar 表示

## 設計上の重要事項

| 仕様書の記述 | 実コードの正しい値 | 理由 |
|---|---|---|
| `IsDeleted` | `DelFlag` | EntityBase の実際のフィールド名 |
| `CreatedBy (long)` | `CreateApplicationUserId (string?)` | EntityBase の実際のフィールド名 |
| 論理削除の実装 | `entity.SetForLogicalDelete()` を呼ぶのみ | 独自フィールド追加不要 |
| `PhycockEntityBase` の場所 | `CommonLibrary/Entity/SiteEntityBase.cs` | ファイル名とクラス名が不一致 |

## 注意事項（実装者へ）

1. **`IsDeleted` は使わない** → `DelFlag` を使用、論理削除は `SetForLogicalDelete()` 呼び出しのみ
2. **`CreatedBy` は使わない** → `CreateApplicationUserId` を使用
3. **`git add -A` 禁止** → 個別ファイル指定（`git add Phycock/Entity/HealthRecordEntity.cs` 等）
4. **秘密情報の直書き禁止** → appsettings.json・コードへの接続文字列・APIキー等の直書き禁止
5. **エラーメッセージ** → 内部情報（DBスキーマ・スタックトレース）をレスポンスに含めない
6. **IDOR対策必須** → サービス層で所有者チェック、コントローラは null → `Forbid()` を返す
7. **マイグレーション確認** → 生成後に DateOnly→`date`型、TimeOnly→`time`型を目視確認してから `database update` 実行
8. **DBContextModelSnapshot は自動更新** → 手動編集禁止
